# 广东综评 MVP 测试 PRD

版本：v0.1
日期：2026-06-04
关联文档：`docs/prd.md`、`docs/frontend-prd.md`
测试范围：前端、后端 API、数据流、管理端、隐私安全、兼容性

## 1. 测试目标

本测试 PRD 用于定义广东综评 MVP 的完整测试范围、测试策略、测试数据、核心用例和验收口径。测试重点是保证学生端查询、时间线、综合分计算器、结构化面经社区，以及管理端审核发布链路稳定、准确、可追溯。

MVP 允许游客浏览核心信息，允许登录用户收藏、投稿、有用、举报，允许管理端完成 AI 草稿、人工审核、发布和风控。测试必须覆盖这些角色边界和数据状态转换。

## 2. 明确不测范围

以下内容不纳入本阶段测试：

- 短信验证码发送、短信供应商、短信到达率、短信模板、短信计费。
- 手机号验证码登录的真实短信链路。
- 微信登录、微信服务号提醒、短信提醒、邮件提醒。
- 小程序、App、支付、课程售卖、志愿填报付费咨询。
- 录取概率预测、录取排名预测、学校热榜。
- 开放评论、私信、社交关系链。

登录相关测试只覆盖：

- 未登录态、已登录态、角色权限态。
- 前端登录入口、回跳和错误展示。
- 后端鉴权拦截、会话失效和权限校验。
- 使用测试环境固定验证码、mock token 或预置测试账号完成登录态验证。

## 3. 测试原则

### 3.1 官方数据优先

- 学生端不得展示草稿、待审核或来源缺失的官方数据。
- 已发布官方数据必须展示学校、年份、来源链接和状态。
- AI 抽取结果必须先进入草稿，不得绕过人工审核直接发布。

### 3.2 工具结果准确

- 综合分计算器必须严格按照已发布公式计算。
- 无明确公式时不得展示计算入口。
- 计算结果不得输出录取概率、录取建议或风险判断。

### 3.3 社区内容可控

- 面经审核前不得进入公共列表。
- 匿名面经不得暴露手机号、认证材料、真实姓名等身份信息。
- 认证材料只允许后台授权角色访问。
- 举报、隐藏、退回、限制账号必须可追溯。

### 3.4 移动体验稳定

- 学生端以移动 Web 为主，375px、390px、430px 宽度必须无横向滚动。
- 卡片、底部导航、表单、筛选、计算结果不得遮挡或溢出。
- 管理端以桌面为主，优先验证表格、抽屉和审核操作。

## 4. 测试类型

### 4.1 单元测试

覆盖范围：

- 综合分公式计算。
- 输入校验。
- 状态计算，如报名状态、节点状态、历史参考。
- 数据脱敏函数。
- 权限判断函数。
- 内容风险检测规则。

通过标准：

- 核心工具函数分支覆盖率不低于 90%。
- 公式计算测试必须覆盖 631、85/15、自定义权重、无公式。
- 隐私脱敏测试必须覆盖手机号、姓名、准考证号、身份证号、认证材料 URL。

### 4.2 API 集成测试

覆盖范围：

- 学生端公开 API。
- 登录用户 API。
- 管理端审核 API。
- 数据状态流转。

通过标准：

- 所有核心 API 在正常、无权限、参数错误、数据不存在场景下返回稳定结构。
- 游客不能访问登录用户和管理端 API。
- 非管理员不能发布简章、公式、面经审核结果。

### 4.3 端到端测试

覆盖范围：

- 高一、高二、高三学生主要路径。
- 管理员官方数据入库到学生端展示路径。
- 学生面经投稿到审核展示路径。
- 计算器使用路径。

通过标准：

- 核心路径无阻塞错误。
- 页面状态和数据状态一致。
- 审核前后学生端可见性正确。

### 4.4 视觉与响应式测试

覆盖范围：

- 学生端主要页面 375px、390px、430px。
- 学生端桌面居中限宽。
- 管理端桌面布局。

通过标准：

- 无横向滚动。
- 底部导航不遮挡主按钮。
- 长学校名、长专业名、长标签可换行或截断。
- 状态标签可读，颜色不是唯一状态表达。

### 4.5 安全与隐私测试

覆盖范围：

- 角色权限。
- 前台敏感字段泄露。
- 认证材料访问控制。
- 面经匿名展示。
- 举报和审核操作审计。

通过标准：

- 前台 API 不返回手机号、认证材料 URL、管理备注。
- 未授权角色无法访问管理端接口和材料文件。
- 所有发布、审核、隐藏、归档操作有操作人和时间。

### 4.6 数据质量测试

覆盖范围：

- 简章字段完整性。
- 2024-2026 年数据范围。
- 来源链接。
- 时间线节点生成。
- 公式来源。

通过标准：

- 每条已发布简章必须有官方来源链接、学校、年份、标题、来源时间。
- 每个已发布公式必须有来源链接和说明。
- 日期未知时展示“待公布”，不得虚构日期。

## 5. 测试环境

### 5.1 环境分层

- 本地开发环境：开发自测、单元测试、组件测试。
- 测试环境：API 集成、端到端、管理端流程、数据状态流转。
- 预发布环境：回归测试、响应式测试、性能冒烟、安全冒烟。
- 生产环境：上线后只做只读冒烟，不创建真实敏感数据。

### 5.2 测试账号

预置角色：

- 游客：不登录。
- 普通学生：可收藏、投稿、有用、举报。
- 内容审核员：可审核面经、认证材料、举报。
- 数据审核员：可审核简章、时间线、公式。
- 管理员：拥有全部管理权限。

登录实现：

- 测试环境使用固定验证码、mock token 或预置会话。
- 不接真实短信，不测试短信发送和到达。

### 5.3 测试数据

基础学校：

- 学校 A：标准 631 公式，2024-2026 均有简章。
- 学校 B：85/15 公式，2025-2026 有简章。
- 学校 C：无明确公式，学生端不展示计算入口。
- 学校 D：当年简章待发布，仅展示历史参考。

简章状态：

- `draft`
- `pending_review`
- `published`
- `archived`

时间线节点：

- 报名未开始。
- 报名进行中。
- 报名即将截止。
- 报名已截止。
- 日期待公布。
- 人工覆盖节点。

面经状态：

- 待审核。
- 已发布。
- 被退回。
- 已隐藏。
- 已认证。
- 未认证。
- 匿名展示。
- 历史参考。

公式配置：

- 631：高考 60%、校测 30%、学考 10%。
- 85/15：高考 85%、校考 15%。
- 自定义权重：多输入项不同满分。
- 无公式：不得计算。

## 9. 数据流转测试

### 9.1 官方简章发布流

步骤：

1. 管理员创建 AI 入库任务。
2. 系统生成来源候选和字段草稿。
3. 数据审核员生成简章草稿。
4. 学生端查询，不可见。
5. 审核员补充来源和关键字段。
6. 发布简章。
7. 学生端可见。
8. 修改简章字段并再次发布。

验收：

- 草稿阶段学生端不可见。
- 发布后学生端可见。
- 版本号递增。
- 来源链接、操作人、发布时间完整。

### 9.2 时间线生成流

步骤：

1. 简章含报名开始、报名截止、校测日期。
2. 发布简章。
3. 系统生成时间线节点。
4. 学生端查看全部节点。
5. 用户收藏学校。
6. 用户查看我的收藏节点。
7. 管理员人工覆盖节点日期。

验收：

- 节点类型正确。
- 收藏时间线只展示收藏学校。
- 人工覆盖原因可追溯。
- 日期未知展示待公布。

### 9.3 面经审核流

步骤：

1. 登录用户提交结构化面经。
2. 投稿进入待审核。
3. 学生端公共列表不可见。
4. 内容审核员通过。
5. 公共列表和学校详情页可见。
6. 用户举报该面经。
7. 管理员隐藏该内容。

验收：

- 审核前不可见。
- 审核后可见。
- 隐藏后不可见。
- 我的投稿始终展示本人投稿状态。

### 9.4 认证材料流

步骤：

1. 用户提交面经并上传认证材料。
2. 认证材料进入审核队列。
3. 普通用户访问材料 URL。
4. 审核员通过认证。
5. 学生端查看面经。

验收：

- 普通用户不能访问材料。
- 前台只展示认证标签。
- 前台不返回材料 URL。
- 审核记录完整。

## 11. 隐私与安全测试

### 11.1 前台字段泄露

检查对象：

- 学校详情。
- 面经列表。
- 面经详情。
- 我的投稿。
- 管理端学生端预览。

不得出现：

- 手机号。
- 认证材料 URL。
- 身份证号。
- 准考证号。
- 真实姓名。
- 后台审核备注。
- 内部风控标签。

### 11.2 匿名展示

用例：

- 匿名面经发布后，前台不展示用户昵称或可识别账号。
- 非匿名面经可展示昵称，但不得展示手机号。
- 匿名状态切换后，前台立即按最新状态展示。

### 11.3 认证材料

用例：

- 游客访问材料 URL 返回 401 或 403。
- 普通学生访问他人材料 URL 返回 403。
- 内容审核员可在审核页访问材料。
- 材料审核通过后前台只展示“已认证”。

### 11.4 内容风控

用例：

- 包含手机号、身份证号、准考证号的面经触发风险提示。
- 包含“包过”“代写”“售卖真题”等内容进入高风险审核。
- 包含具体未公开原题的内容需要退回改写。

## 16. 缺陷分级

### P0

- 草稿或待审核官方数据公开展示。

- 非管理员可发布简章、公式或审核内容。

- 面经审核前公开展示。
- 时间线关键截止日期展示错误。
- 无公式学校展示计算入口。
- 管理端审核操作失败。
- 移动端核心页面严重遮挡或无法操作。

- 筛选、排序、空状态异常。
- 页面局部样式不符合规范。
- 非核心字段展示不完整。
- toast 或错误文案不清晰。

- 轻微文案问题。
- 非核心视觉细节偏差。
- 低频页面布局瑕疵。
