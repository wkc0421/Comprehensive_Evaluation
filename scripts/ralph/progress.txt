## Codebase Patterns
- The app scaffold is dependency-free Node.js ESM; keep routing in `src/app.js` and server startup in `src/server.js`.
- Server-rendered HTML is composed in `src/pages.js`, shared product metadata is in `src/lib/product.js`, and global mobile-first CSS is in `public/styles.css`.
- Student-facing HTML should use the shared student shell helpers in `src/pages.js`; keep the four-tab Home/Schools/Experiences/My bottom nav in `src/lib/product.js`, use page-type top bars, and hide the bottom nav only on task pages with a clear back entry.
- Admin-facing HTML should use the shared desktop admin shell helpers in `src/pages.js`; keep Data Ingestion/Guide Review/Timeline Management/Formula Management/Experience Review/Verification Review/Report Handling navigation in `src/lib/product.js`, and verify admin UI changes through the admin desktop section of `scripts/browser-core-pages.mjs`.
- Quality scripts live in `scripts/` and intentionally avoid scanning Ralph run logs or source PRD docs.
- MVP-wide quality gates live in `src/e2e.test.js`, `scripts/data-quality.mjs`, and `scripts/browser-core-pages.mjs`; run `npm test`, `npm run data-quality`, and `npm run browser-test` for end-to-end acceptance.
- `npm run build` creates `dist/src` and `dist/public`; `npm start` runs `dist/src/server.js`, and `dist/` is generated and ignored.
- Core PostgreSQL schema migrations live in `src/db/migrations`; `src/db/schema.test.js` validates migration SQL statically without connecting to `DATABASE_URL`.
- Seed fixtures and student-facing published-only data access helpers live in `src/db` as plain JavaScript modules; use them before introducing database tooling.
- Authentication/session helpers live in `src/auth.js`; `handleRequest` accepts an optional `{ authService }` test context, and public auth responses must use the public user shape so phone fields do not leak.
- `/logout` and `/api/logout` destroy the current session through `authService.destroySession()` and set a clear-session cookie through `serializeClearSessionCookie()`; client-side student draft cleanup hooks onto the logout form.
- Student-facing pages should consume published-only helpers from `src/db/data-access.js` instead of importing `seed-data.js` directly.
- Public school browsing should use `listSchoolGuideCards()` from `src/db/data-access.js`; it keeps `/schools`, `/api/schools`, and the HTML school list aligned on published-only guides, filters, sort order, key timeline nodes, formula availability, and experience signals.
- Public school detail pages and APIs should use `getSchoolDetail()` from `src/db/data-access.js`; it keeps `/schools/:id`, `/api/schools/:id`, year defaulting, published-only visibility, timeline/formula aggregation, and featured experience ranking aligned.
- Public school records include `abbreviation` in the seed/public JSON shape, and public keyword matching should include both school name and abbreviation.
- School detail HTML may fall back from an unpublished requested/current year to the latest published guide with a historical-reference label; `/api/schools/:id?year=...` should keep returning a published-only 404 for unpublished years.
- The school list AJAX filter enhancement in `public/student.js` swaps server-rendered filter/results sections marked by `data-school-filters-container` and `data-school-results-section`; keep no-JS GET form behavior intact and verify failed-load retry UI in browser tests.
